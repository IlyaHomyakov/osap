Создание текста

В AutoCAD можно задавать различные типы текста (стили). Их задание обеспечивает команда STYLE, которая выводит диалоговое окно Text Style.

Меню

Annotate | Text

Командная строка

STYLE


В выпадающем списке Style Name можно выбрать необходимое имя стиля, для создания нового имени следует использовать кнопку New. Для изменения шрифта данного стиля используется выпадающий список Font name. Шрифт определяет форму текстовых символов. В AutoCAD можно использовать шрифты True Type в дополнение к собственным шрифтам (SHX). Многострочный текст, создаваемый командой МТЕХТ, отображает шрифты, поддерживаемые Windows. Поскольку шрифты SHX AutoCAD не поддерживаются Windows, AutoCAD содержит эквивалент True Type при выборе шрифтов SHX или других шрифтов, не являющихся True Type.

AutoCAD поддерживает Unicode character-encoding стандарт. Шрифт Unicode содержит 65.535 символов для многих языков. Можно использовать символы, которые не доступны непосредственно с клавиатуры, введя последовательность U+nnnn, где nnnn - шестнадцатеричный код символа. Все шрифты SHX представлены в стандарте Unicode.

Для кастомизированных шрифтов SHX или шрифтов, разработанных третьими фирмами и не имеющих эквивалента True Type, AutoCAD поставляет до восьми различных True Type шрифтов, называемых шрифтами-заместителями (proxy fonts). В связи с этим возможно несоответствие изображения текста в окне команды МТЕХТ и на реальном рисунке.

Height - высота текста. Для шрифтов True Type высота не представляет собой высоту самой высокой буквы, а задает высоту области, включающей букву и другие значки нелатинского алфавита. Если принять высоту символов равной 0, то будет выдаваться запрос о высоте символов в командах TEXT и DTEXT.

Width factor - Степень сжатия-растяжения <по умолчанию>: - масштабный коэффициент.

Obliquing angle -угол наклона - наклон текста по отношению к нормали: положительный - вправо (по часовой стрелке), отрицательный - влево (против часовой стрелки).

Backwards - задание текста в обратном порядке.

Upside-down - задание перевернутого текста.

Vertical - задание написания букв одна над другой.

Вертикальная ориентация задается для шрифтов, поддерживающих двойную ориентацию. Нормальным углом поворота для вертикально ориентированного текста является угол в 270°. Многострочный текст можно вводить и в вертикальной ориентации, при этом каждая последующая текстовая строка выводится справа от предыдущей. Угол наклона, подчеркивание и надчеркивание использовать с вертикально ориентированным текстом нельзя.

При создании нового рисунка подразумевается стандартный стиль:

Имя текстового стиля STANDARD
Имя файла шрифта txt
Высота 0
Степень сжатия-растяжения 1
Угол наклона 0
Справа налево Нет
В перевернутом виде Нет
Ориентация Горизонтальный

Если характеристики текстового стиля изменяются, то это никак не влияет на существующие текстовые примитивы, созданные с использованием данного стиля. Однако если задать другой файл шрифта или другую ориентации текста, то весь текст, использующий данный стиль, регенерируется с использованием нового шрифта или ориентации. Это происходит немедленно, если только не отключен режим Регенавто. Для изменения имени существующего текстового стиля можно использовать команду RENAME. Если в каком-либо из существующих текстовых элементов используется старое имя стиля, то замена имени осуществляется автоматически.

Для включения в чертеж простого текста используются команды ТЕХT, DTEXT.

Для включения в чертеж одной строки текстовой информации в AutoCAD используется команда TEXT.

Командная строка

TEXT

Для включения в чертеж нескольких строк текстовой информации в AutoCAD используется команда DTEXT.

Командная строка

DTEXT

Запросы для команд ТЕХT и DTEXT идентичны:

Justify/Style/<Start point>:
Выравнивание/Стиль/<Начальная точка>:

В ответ на запрос необходимо:

  • Указать точку - для выравнивания строки текста по левой точке вдоль базовой линии.
  • Ввести J - для выравнивания текстовой строки с использованием ключей выравнивания.
  • Ввести S - для установки нового текстового стиля.
  • ENTER или Пробел для перевода каретки и размещения текстовой строки непосредственно под предыдущей.

Запросы для текста с указанной начальной точкой:

Height <2.5000>:

Определяет высоту прописных букв над базовой линией. По умолчанию принимается высота, которая использовалась предыдущим текстовым элементом с тем же текстовым стилем. Запрос выдается, если в стиле задано значение высоты, равное нулю, в противном случае берется значение высоты из текстового стиля.

Rotation angle <0>:

Задает ориентацию базовой линии текста относительно начальной точки. По умолчанию используется последнее заданное значение угла.

Text:

Запрашивает ввод текстовой строки, при этом допускается ввод пробелов.

Если при формировании выровненного текста сразу известно, какое выравнивание будет использоваться, можно вместо J (выравнивание) и ввести сразу его имя. Дальнейший диалог с системой зависит от ориентации. При горизонтальной и вертикальной ориентации стиля текста выдаются соответственно следующие запросы:

Align/Fit/Center/Middle/Right/TL/TC/TR/ML/MC/MR/BL/BC/BR:

Ключи:

Align (вписанный) - служит для задания начальной и конечной точек текста, высота и ширина каждого символа автоматически вычисляется системой AutoCAD так, чтобы текст точно вписывался в заданную область.
Fit (по ширине) - служит для вывода строки заданной высоты, вписанной между двумя точками.
Center (центр) - обеспечивает центрирование базовой линии относительно заданной точки.
Middle (середина) - обеспечивает горизонтальное и вертикальное центрирование текста относительно заданной точки. Различие между данным ключом и ключом МС состоит в том, что используется не средняя точка между верхом и базовой линией, а середина воображаемой рамки, в которую взята строка текста. Таким образом, различие определяется наличием символов, доходящих до нижней или верхней линии.
Right (правосторонний) - служит для вывода текста, выровненного по правому краю.

Ключи выравнивания и их описание:

TL - текст выровнен вверх влево.
ТС - текст выровнен вверх по центру.
TR - текст выровнен вверх вправо.
ML - текст выровнен посередине влево.
МС - текст центрирован по вертикали и по горизонтали в средней точке.
MR - текст выровнен посередине вправо.
BL - текст выровнен вниз влево.
ВС - текст выровнен вниз по центру.
BR - текст выровнен вниз вправо.

По умолчанию текст выравнивается по левому краю.

Для всех случаев задания текста, кроме Align и Fit, запрашивается угол поворота базовой линии:

Rotation angle <0>:

При использовании всех ключей, кроме Align, запрашивается высота текста:

Height <2.5000>:

Во всех случаях ввод текста осуществляется в ответ на запрос Text:

Для создания многострочного текста необходимо нажать ENTER в ответ на Command: (команда:) для повторения команды и ввести ENTER в ответ на запрос Start point: (начальная точка:). При этом система AutoCAD предполагает, что продолжается задание текста непосредственно под введенной перед этим строкой с использованием тех же самых параметров: угла поворота, слоя, текстового стиля, цвета, высоты и типа выравнивания.

Некоторые шрифты поддерживают управляющие коды и специальные символы, которые можно задать путем включения в текстовую строку управляющей информации в виде управляющих последовательностей, каждая из которых начинается с двух символов процента " % % ". Например, возможны следующие управляющие последовательности:

% % о - переключение режима надчеркивания (Вкл/Откл);

% %u - переключение режима подчеркивания (Вкл/Откл);

%%d- специальный символ "градус";

% %р - специальный символ "допуск";

%%с- специальный символ "диаметр";

% % % - вывод единичного символа процента;

% % nnn - специальный символ с десятичным кодом nnn.

Заметим, что надчеркивание и подчеркивание могут действовать одновременно. Оба режима при достижении конца текстовой строки автоматически отключаются.

Для ввода нескольких строк текста может служить команда DTEXT - динамический текст, в которой используются такие же ключи, что и в команде TEXT.

Запрос Text: выдается циклически. Строки текста выводятся последовательно одна под другой. Для окончания ввода текста необходимо нажать ENTER.

При использовании команды DTEXT на экране появляется курсор, соответствующий размерам символа текста. В процессе набора текста на клавиатуре он отображается на графическом экране.

Во время ввода текста графический маркер можно перемещать независимо от текстового курсора. Если при вводе текста указать на экране некоторую точку с помощью графического маркера, то команда DTEXT завершит ввод текущей строки и переместит текстовый курсор в указанную точку. Таким образом можно размещать текст в разных точках экрана.

Команда DTEXT позволяет вводить управляющие коды и специальные символы. Однако управляющие последовательности заменяются символами только по завершении команды DTEXT. Это позволяет редактировать управляющие последовательности.


Создание многострочного текста

Кнопка инструментов
Multiline Text
     

Меню

Annotate | Text | Multiline Text

Командная строка

MTEXT

Запросы:

Specify first corner:
Выберите первый угол:
Specify opposite corner or [Height/Justify/Rotation/Style/Width]:
Выберите противоположный угол или [Высота/Выравнивание/Поворот/Стиль/Ширина]:

После этого появляется диалоговое окно Multiline Text Editor (Редактор многострочного текста):


В этом окне текст вводится как в обычном текстовом редакторе, кроме того, с помощью кнопок на верхней панели инструментов можно устанавливать атрибуты шрифта, такие как подчеркивание, жирный, курсив, размер. Шрифт выбирается из первого выпадающего списка на странице Character. В следующем выпадающем списке можно выбрать размер шрифта. Правда если необходимо установить ранее не использовавшийся размер, его надо ввести с клавиатуры, т. к. в списке присутствуют только ранее использовавшиеся величины. Можно импортировать текст из текстового файла с помощью кнопки Import Text. Завершается редактирование и вставляется текст в чертеж с помощью кнопки ОК.


Редактирование текста

Кнопка инструментов
Edit Text
     

Командная строка

DDEDIT

Запрос:

<Select an annotation object>/Undo:
<Выберите редактируемый объект>/Отмена:

После выбора редактируемого объекта появится окно текстового редактора.
