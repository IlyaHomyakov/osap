4. Создание Таблицы Управления Деталями
(Creating Table Driven Parts)

Можно назначать переменные на параметрические размеры, которые управляют базовой деталью, и затем использовать электронную таблицу для контроля размера и формы детали. Электронная таблица может содержать несколько версий детали.

Каждая версия использует разные значения для определенных вами переменных. Mechanical Desktop перерисовывает деталь, используя переменные, заданные для текущей версии. Документация для всех версий может быть упрощена созданием одного чертежа, представляющего универсальную деталь.

Электронная таблица вставлена в чертеж для регистрации значений переменных каждой версии.

Для использования этого руководства вы должны установить программный продукт фирмы Microsoft для работы с электронными таблицами Microsoft Excel на компьютере.


4.1. Ключевые термины


Термин

Определение

Переменная активной детали (Active part variable)

Параметрическая переменная, используемая в размерах, которые управляют параметрами активной детали

Подавление (гашение) опций (Feature suppression)

Опция, которая подавлена или временно удалена из учета детали. Опции могут быть подавлены вручную или из внешней таблицы.

Глобальная переменная (Global variable)

Параметрическая переменная, которая может использоваться любым количеством опций и деталей.

Спиральная развертка (Helical sweep)

Геометрическая опция, определяемая величиной от перемещения профиля по трехмерному пути вокруг рабочих осей.

Связанная электронная таблица (Linked spreadsheet)

Внешняя электронная таблица, динамически связанная с чертежом. Любые модификации в электронной таблице непосредственно затрагивают чертеж.

Ручное подавление (Manual suppression)

Опции, подавленные в окне просмотра. Опции, подавленные вручную, подавляются во всех версиях детали.

Версия детали (Part version)

Версия таблицы, управляющей деталью, которая определена в связанной электронной таблице.

Шаг (Pitch)

Мерная линия, параллельная оси спирального пути, от одной точки на пути до соответствующей точки на смежном обороте.

Профильная плоскость (Profile plane)

Рабочая плоскость в начальной точке спирального пути, расположенная перпендикулярно началу пути или в центре оси/пути.

Начальный угол (Start angle)

Угол, под которым начинается спиральный путь от оси x активного плана эскиза.

Таблица (Table)

Внешняя электронная таблица, которая управляет версиями детали.

Таблица управления подавлением (Table driven suppression)

Функции, подавленные во внешней электронной таблице. Таблица управления подавлением детали подавляет только версию детали, определенную в электронной таблице.

Таблица управления переменными (Table driven variable)

Глобальная или локальная переменные детали чертежа контролируются значениями внешней электронной таблицы.

Угол конусности (Taper angle)

Угол, под которым спиральная развертка сходится к вершине конуса.


4.2. Руководство быстрым взглядом


4.2.1. Что нового в этом руководстве

Таблицы управления деталью были введены в предыдущей версии Mechanical Desktop. Следующие нововведения и расширения вы найдете в данном руководстве:

  • Спиральная развертка
  • Таблица управления глобальными расчетными параметрами
  • Расчетными параметрами активной детали
  • Подавление опций
  • Таблица управления подавления опций
  • Разрешение противоречий с помощью электронных таблиц

Остальные возможности включают:

  • Использование браузера для выполнения большинства задач
  • Использование панели Вид (View) для контроля расположения деталей
  • Использование кнопки инструментов, меню или командной строки для доступа к командам

Для дополнительной информации о нововведениях и расширениях смотрите часть 1 описания Подготовка к работе (Getting Started).


4.2.2. Основные сведения о таблице управления деталями

В производственной индустрии много деталей, отличающихся только размерами или имеющих небольшие различия в форме. Например, пружины, кронштейны, шайбы, гайки, болты. С помощью управления деталью с внешней электронной таблицы можно документировать число аналогичных деталей на одном чертеже.

Все конструкторские данные сводятся в одну таблицу, на одном чертеже представляется много деталей.

4.4. Использование переменных чертежа

Переменные чертежа представляют средство контроля над размерами, используя связи между ними. Изменение одной или более переменных отражается на детали в целом.

Переменные чертежа могут быть локальными и глобальными.


4.4.1. Локальные переменные чертежа

Локальные переменные чертежа управляют только деталью, которая с ними связана. Используйте локальные переменные чертежа для управления версиями детали при помощи таблицы.


4.4.2. Глобальные переменные чертежа

Глобальные переменные чертежа также могут действовать из таблицы. Это позволяет вам использовать одни и те же переменные для различных операций над различными деталями. Если вы проектируете много деталей в одном файле, то можно использовать глобальные переменные для управления несколькими или всеми деталями с одинаковыми переменными.


4.5. Создание локальных переменных детали

Спиральная развертка управляется следующими переменными:

  • Тип развертки (Type of sweep)
  • Количество оборотов (Number of revolutions)
  • Шаг (Pitch)
  • Высота (Height)
  • Форма (Shape)
  • Диаметр развертки (Diameter of the sweep)
  • Конусный угол (Taper angle)
  • Ориентация (Orientation)
  • Стартовый угол (Start angle)
  • Радиус развертывающего профиля (Radius of the swept profile)

В этом руководстве вы создадите переменные и параметрические уравнения для управления количеством оборотов, высотой и диаметром развертки. Вы также назначите переменную для управления радиусом профиля, который развертывается вдоль спирального пути. Так как вы работаете с одной деталью, то будете создавать локальные переменные детали.

Позднее вы создадите электронную таблицу для определения четырех версий этой детали, изменяя значения локальных переменных для каждой версии.

4.12. Подавление(гашение) опций

Подавление опций выполняется из браузера или из связанной таблицы. Подавление из браузера отражается на всех версиях детали. Таблица позволяет контролировать каждую версию.

В созданную таблицу добавляем элементы, которые мы хотим подавлять. Сначала вручную подавляем стяжку, и это изменение отобразится в браузере для всех версий детали. При этом не важно, какая версия детали активна в данный момент.
